# 历史文档归档

本文回答：`docs/_archive/` 里放什么、哪些内容已经退出主阅读路径，以及迁移到现行文档前应如何使用这些历史资料。

## 30 秒结论

- `docs/_archive/` 只保存**历史文档**和**旧体系正文**，不再属于主阅读路径。
- 归档目录默认保留旧稿原貌；现状版正文应写在新目录体系里，不在 `_archive/` 内继续演化。
- 当前已经归档的是：
  - `00-概览/00-文档体系重组方案.md`
  - `00-概览/03-grpc服务设计.md`
  - `00-概览/04-qs接入iam指南.md`
  - `00-概览/05-核心优先迁移清单.md`
  - `suggest.md`
  - `01-认证域/`
  - `02-授权域/`
  - `03-用户域/`
  - `04-基础设施/`
  - `ops/`
- 这些文档可以继续作为术语、图示、历史设计和迁移参考的信息源，但**不能默认视为当前实现**。
- 任何内容如果要重新进入现行正文，必须重新对照源码、契约、配置和运行时装配。

## 使用规则

1. `docs/_archive/` 默认不参与现行文档的结构要求与卫生检查。
2. 归档文档可以引用，但引用时应明确说明它是“历史资料”。
3. 现行文档优先链接到新框架；只有在确实需要解释历史背景时，才回链到 `_archive/`。

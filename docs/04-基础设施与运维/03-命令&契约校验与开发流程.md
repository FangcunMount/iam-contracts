# 命令、契约校验与开发流程

本文回答：`iam-contracts` 当前有哪些值得直接记住的 `make` 目标、它们分别对应什么真实脚本或目录，以及日常开发、契约校验和 Docker 交付应如何组织。

## 30 秒结论

- 当前仓库的命令面以 [../../Makefile](../../Makefile) 为中心，最重要的是 `build / run / stop / restart / status / logs / health`、测试质量链、契约生成校验链和 Docker / 数据库辅助命令。
- 当前服务面实际上围绕单个 `iam-apiserver` 组织，因此聚合目标和 `*-apiserver` 目标本质上是同一件事。
- 契约相关改动最应该记住的是：`make docs-swagger`、`make docs-reset`、`make api-validate`、`make proto-gen`。
- 如果要切到开发配置，本地运行时不要忘了覆盖 `APISERVER_CONFIG`，因为默认值是 `configs/apiserver.prod.yaml`。
- 证书相关目标里，`grpc-cert-verify` / `grpc-cert-info` 目前可信；`cert-gen` / `cert-test` / `test-dev-config` 仍引用缺失脚本，应视为遗留项。

## 1. 当前命令分层

### 1.1 服务生命周期

| 目标 | 当前作用 | 备注 |
| ---- | ---- | ---- |
| `make build` | 构建所有服务 | 当前实际落到 `build-apiserver` |
| `make build-apiserver` | 构建 `bin/apiserver` | 入口是 `cmd/apiserver/` |
| `make run` | 启动服务 | 当前实际落到 `run-apiserver` |
| `make run-apiserver` | 后台启动 `iam-apiserver` | 默认配置是 `configs/apiserver.prod.yaml` |
| `make stop` | 停止服务 | 当前实际落到 `stop-apiserver` |
| `make restart` | 重启服务 | 当前实际落到 `restart-apiserver` |
| `make status` | 查看状态 | 当前实际落到 `status-apiserver` |
| `make logs` | 查看日志 | 当前实际落到 `logs-apiserver` |
| `make health` | 健康检查 | 当前实际落到 `health-check` |

### 1.2 开发 / 测试 / 质量

| 目标 | 当前作用 |
| ---- | ---- |
| `make dev` / `make dev-apiserver` | Air 热更新开发模式 |
| `make dev-stop` / `make dev-status` / `make dev-logs` | 管理开发环境 |
| `make test` / `make test-unit` / `make test-coverage` / `make test-race` / `make test-bench` | 测试与覆盖率 |
| `make lint` / `make fmt` / `make fmt-check` | 代码质量 |
| `make ci` | `deps-verify + fmt-check + lint + test` |

### 1.3 契约 / 文档 / 生成

| 目标 | 当前作用 |
| ---- | ---- |
| `make docs-swagger` | 重新生成 `internal/apiserver/docs/swagger.yaml` |
| `make docs-reset` | 从 swagger 重置 `api/rest` |
| `make api-validate` | OpenAPI lint + swagger 与 `api/rest` 比对 |
| `make proto-gen` | 生成 protobuf 代码 |

### 1.4 数据 / Docker / 部署辅助

| 目标 | 当前作用 |
| ---- | ---- |
| `make db-bootstrap` / `make db-status` / `make db-backup` / `make db-connect` | MySQL 辅助 |
| `make docker-mysql-*` | 本地开发 MySQL 容器 |
| `make docker-build` / `make docker-run` / `make docker-stop` / `make docker-clean` / `make docker-push` | 镜像与单容器管理 |
| `make docker-compose-*` | 生产 compose |
| `make docker-dev-*` | 开发 compose |
| `make deploy-check` | 检查部署变量 |

## 2. 最常用的工作流

### 2.1 本地开发

```bash
make help
make install-tools
make dev
make fmt
make test
make ci
```

### 2.2 本地运行指定配置

```bash
make run APISERVER_CONFIG=configs/apiserver.dev.yaml
make status
make logs
```

### 2.3 契约相关改动

```bash
make docs-swagger
make docs-reset
make api-validate
make proto-gen
```

### 2.4 Docker 与数据库

```bash
make docker-dev-up
make docker-dev-logs
make db-bootstrap DB_PASSWORD=your_password
```

## 3. 环境变量与默认值

以下变量直接来自 `Makefile`，是当前文档最值得记住的一组默认值：

| 变量 | 默认值 | 当前含义 |
| ---- | ---- | ---- |
| `APISERVER_BIN` | `bin/apiserver` | apiserver 二进制输出路径 |
| `APISERVER_CONFIG` | `configs/apiserver.prod.yaml` | `make run` 默认配置 |
| `APISERVER_DEV_CONFIG` | `configs/apiserver.dev.yaml` | 开发配置参考值 |
| `APISERVER_PORT` | `8080` | `status` / `ports` / `health-check` 使用的 HTTP 端口 |
| `APISERVER_SSL_PORT` | `8443` | `health-check` 使用的 HTTPS 端口 |
| `PID_DIR` | `tmp/pids` | 进程 PID 目录 |
| `LOG_DIR` | `logs` | 本地日志目录 |
| `COVERAGE_DIR` | `coverage` | 覆盖率报告目录 |

## 4. 当前边界

### 已实现

- 聚合服务目标和 `*-apiserver` 目标当前已对齐
- swagger / OpenAPI / proto 的生成与校验链
- Air、Docker、数据库辅助命令
- `grpc-cert-verify` / `grpc-cert-info` 与 infra 统一证书路径对齐

### 待清理

以下目标当前仍存在，但不能写成稳定入口：

| 目标 | 当前问题 |
| ---- | ---- |

| `make cert-test` | 依赖缺失的 `scripts/cert/test-cert.sh` |
| `make test-dev-config` | 会间接调用缺失的本地证书脚本 |

当前更可信的证书说明应以 [../../scripts/cert/README.md](../../scripts/cert/README.md) 和 `make grpc-cert-*` 为准。

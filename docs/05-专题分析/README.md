# 专题分析

本文回答：`iam-contracts` 里哪些内容适合单独拉成跨层专题来讲，这一组文档和 `业务域 / 接口与集成 / 运行时` 如何分工，以及当前应该先看哪条主链路。

## 30 秒结论

- `05-专题分析` 负责讲“跨层主链路”和“关键设计选择”，不替代业务域正文，也不替代 `api/` 契约。
- 这里更适合放“认证链”“授权判定链”“监护关系链”这类需要同时回链 interface / application / domain / infra 的深挖文档。
- 当前已经补的是认证主链、授权判定链、监护关系链，以及 SDK 封装与接入价值专题。

# 接口与集成

本文回答：`iam-contracts` 的接口解释层应该从哪里读起，`docs/` 与 `api/` 之间如何分工，以及当前有哪些对外接入文档已经迁入这一层。

## 30 秒结论

- `api/` 是机器契约层，`docs/03-接口与集成` 是对这些合同和接入方式的解释层。
- 这一组文档回答“怎么接、看哪份合同、验证什么”，不替代 `OpenAPI` 或 `Proto` 本身。
- 当前这一组已经有 REST、gRPC、授权接入边界、身份接入边界，以及面向 `qs` 的接入说明；跨层主链路则由 `05-专题分析` 承载，认证链、授权链和监护关系专题都已经建立。
